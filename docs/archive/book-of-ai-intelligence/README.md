# Book of AI Intelligence

Companion reference for how those ideas map onto real AI-lab practice and Spark's own architecture:

- [AI Lab Map](../AI_LAB_MAP.md)

*A nontechnical training playbook for building a smarter agent.*

This folder is written as a readable book, not as lab notes.

It is for people who want to build AI systems that are:

- more useful than theatrical
- more thoughtful than merely fluent
- more focused than bloated
- more reliable than benchmark-polished

The core idea is simple:

An AI does not become intelligent just because it has seen more words.
It becomes more intelligent when it learns how to notice, compare, check, remember, specialize, and improve without drifting into nonsense.

## What This Book Tries To Do

This book turns a large ruleset into plain English.

It explains:

- what makes AI primitive versus genuinely intelligent
- how trainers avoid wasting time on low-value learning
- how memory should work without becoming clutter
- why small clean systems often beat swollen complex ones
- how self-improving agents should be governed
- why benchmark scores can mislead builders
- how to make an AI specialized and strong at one job
- how to keep progress aligned with human goals
